1. Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design? 

Our Client, the Gaming Room, Was wanting to expand their already existent game Draw It or Lose It into a web‑based, multi‑platform application. The goal behind this was to support multiple teams, multiple players, and real‑time gameplay across different operating systems.This meant they needed a scalable, secure, and distributed architecture that could run on various devices while maintaining consistent performance and user experience.

5. How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing? 

I interpreted user needs by focusing on what the players and game hosts required: fast performance, real‑time updates, cross‑platform access, and a smooth interface. These needs shaped decisions like using a distributed architecture and ensuring scalability. Considering user needs is essential because the design must support how the software will actually be used. Ignoring user needs leads to poor usability and features that don’t solve real problems.
